Page 3 — Owner Permissions

Owner has exclusive access to critical bot commands and settings.
Commands like restart, reload, eval, and config management are owner-only.
The bot recognizes the owner by the Discord ID set in your config.
>ᴗ<
Page 4 — Setting Owner ID

To set or update your owner ID:

Open the .env or configuration file.
Locate the owner ID entry, for example:
OWNER_ID=123456789012345678
Replace with your Discord user ID.
Save and restart your bot.
>ᴗ<
Page 5 — Owner-Only Commands

Typical owner-only commands include:

Restart bot
Reload commands
Evaluate JavaScript code
Set bot status/presence
Modify bot configuration
>ᴗ<
Page 6 — Security Best Practices

Keep your owner ID private.
Do not share your .env or sensitive config files.
Use environment variables for all secrets (tokens, keys, IDs).
Never commit secrets to public repositories.
>ᴗ<
Page 7 — Backup & Recovery

Regularly backup your configuration files.
Document any changes to the bot setup.
In case of issues, you can restore from backups quickly.
>ᴗ<
Page 8 — Updating the Bot

Keep dependencies and libraries up to date.
Test updates in a separate environment before deploying live.
Use version control (Git) to manage your code.
>ᴗ<
Page 9 — Support & Contribution

Contact the owner directly for support.
Contributions can be done via pull requests or issues if using a repository.
Maintain clear communication about changes or bugs.
